### 3.3 API 架构设计

```
┌─────────────────────────────────────────────────────────┐
│                    ApiProvider (接口)                     │

│  │  fun sendMessageStream(                             ││
│  │    messages: List<Message>,                         ││
│  │    systemPrompt: String,                            ││
│  │    onSuccess: (String) -> Unit,     流式增量回调     ││
│  │    onComplete: (String) -> Unit,    完成回调         ││
│  │    onError: (String) -> Unit        错误回调         ││
│  │  )                                                  ││
│  └─────────────────────────────────────────────────────┘│
│                          △                              │
│                          │ 实现                          │
│  ┌─────────────────────────────────────────────────────┐│
│  │          OpenAiCompatibleProvider                    ││
│  │  - 使用 OkHttp 发送 POST 请求                        ││
│  │  - 使用 SSE 解析流式响应 (data: ...)                 ││
│  │  - 兼容所有 OpenAI Chat Completions 格式接口         ││
│  │  - 自动处理完成标记 [DONE] 和 finish_reason          ││
│  │  - 统一错误解析（401/402/429/500/503）              ││
│  └─────────────────────────────────────────────────────┘│
│                                                          │
│  ┌─ ApiConfigManager ───────────────────────────────────┐│
│  │  - 单例模式 (getInstance)                             ││
│  │  - 预置提供商列表 (builtInProviders)                  ││
│  │  - SharedPreferences 持久化用户选择 + API Key        ││
│  │  - createCurrentProvider() 创建当前选中的提供商实例   ││

│  └─────────────────────────────────────────────────────┘│
└─────────────────────────────────────────────────────────┘
```

### 3.5 OpenAI 兼容 API 请求/响应格式

**请求** (POST):
```json
{
  "model": "deepseek-ai/DeepSeek-V2.5",
  "messages": [
    {"role": "system", "content": "你是一个专业的股票投资助手..."},
    {"role": "user", "content": "用户的问题"}
  ],
  "stream": true,
  "temperature": 0.7,
  "max_tokens": 2048
}
```

**响应** (SSE 流式):
```
data: {"choices":[{"delta":{"content":"第"}}]}
data: {"choices":[{"delta":{"content":"一"}}]}
data: {"choices":[{"delta":{"content":"步"}}]}
data: {"choices":[{"delta":{"content":""},"finish_reason":"stop"}]}
data: [DONE]
```

## 6. API Key 获取与配置

### 6.1 方式一：硅基流动（推荐，免费）

1. **注册**：电脑浏览器访问 https://cloud.siliconflow.cn/ （手机号注册）
2. **获取 API Key**：登录 → API Keys 页面 → 创建 Key
3. **在 App 中配置**：
   - 打开 AI 聊天 → 点击顶部 **「API」** 按钮
   - 选择 **「🆓 硅基流动 (免费)」**
   - 在弹出的输入框中粘贴你的 API Key
   - 点击「保存」即可使用 🎉

> 硅基流动免费额度：每月数百万 tokens，个人日常使用完全足够。

### 6.2 方式二：DeepSeek 官方（付费）

1. **注册**：电脑浏览器访问 https://platform.deepseek.com/api_keys
2. **获取 API Key**：登录 → API Keys → 创建 Key
3. **在 App 中配置**：同上，选择「💳 DeepSeek 官方」

> 费用：新用户送 500 万 tokens，之后约 0.5 元/百万 tokens。

### 6.3 方式三：阿里云百炼（需自行添加）

在 `ApiConfigManager.kt` 的 `builtInProviders` 列表中添加：

```kotlin
ApiProviderConfig(
    id = "dashscope",
    name = "阿里云百炼",
    baseUrl = "https://dashscope.aliyuncs.com/compatible-mode/v1/chat/completions",
    apiKey = "",  // 用户输入
    model = "qwen-plus",  // 或 deepseek-r1 等
    description = "阿里云百炼平台，通义千问/DeepSeek 模型",
    isFree = false
)
```

### 6.4 API Key 管理机制

```
用户输入 API Key
       │
       ▼
ApiConfigManager.saveApiKey(providerId, key)
       │
       ▼
SharedPreferences 持久化存储 (key: "api_key_{providerId}")
       │
       ▼
启动 App → ApiConfigManager.createCurrentProvider()
       │
       ▼
从 SharedPreferences 读取 API Key → 创建 OpenAiCompatibleProvider 实例
```
