# prefetch/ — 后台数据预热调度

## 文件

| 文件 | 说明 |
|------|------|
| `StockPrefetchScheduler.kt` | 预取调度器：按历史高频股票/主题周期预热缓存 |
| `UserQueryHistory.kt` | 用户查询历史记录（SharedPreferences 持久化） |

## 调度策略

```
App 启动 30s 后开始
  ├─ 活跃时段 (9:00-15:30 + 晚间) → 每 5 分钟预热
  └─ 非活跃时段 → 每 30 分钟预热

预热目标：
  ├─ 高频股票代码（取前10只）
  └─ 高频主题（取前3个，每个取前10只股票）
```

预热数据写入 SmartStockCache，TTL=10分钟。用户发问时缓存命中秒返回。
