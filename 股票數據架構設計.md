# 股票數據服務架構設計

## 1. 問題分析

### 1.1 用戶輸入的多樣性

我們需要處理的用戶輸入場景：

| 場景 | 示例 | 處理方式 |
|------|------|----------|
| **直接股票代碼** | "600519" | 正則匹配，直接查詢實時行情 |
| **股票名稱** | "茅台"、"貴州茅台" | 名稱→代碼映射表查詢 |
| **模糊名稱** | "茅臺"（錯別字）、"茅臺酒" | 先問 AI 糾正，再查詢 |
| **多隻股票** | "茅台和五糧液"、"茅台,平安,寧德時代" | 批量查詢多隻 |
| **指數查詢** | "上證指數"、"創業板指" | 特殊處理（行情格式不同） |
| **AI 需要理解的查詢** | "今天哪些股票漲停了？"、"白酒板塊走勢如何"、"推薦幾隻潛力股" | 先調用 AI 解析意圖，再執行對應操作 |
| **技術分析** | "分析茅台的MACD"、"畫一下茅台的K線" | 需要調用歷史數據接口 |
| **市值/基本面** | "茅台的市值多少"、"比亞迪的PE" | 需要調用財務數據接口 |
| **比較分析** | "茅台和五糧液哪個更值得買" | 多隻股票對比，AI 綜合分析 |

### 1.2 現有方案的問題

當前的 `StockDataProvider` 只做了最簡單的實時行情查詢：
- 只能處理簡單的股票代碼/名稱
- 沒有意圖識別（無法區分「查股價」vs「熱門股票」）
- 沒有多數據源支持
- 沒有緩存
- 沒有錯誤恢復
- 無法處理複雜查詢

## 3. 核心組件設計

### 3.1 意圖識別層（IntentProcessor）

使用 **Chain of Responsibility 設計模式**，每個處理器負責一種意圖：

```kotlin
// 意圖枚舉
enum class StockIntent {
    QUERY_PRICE,        // 查詢實時股價（單隻/多隻）
    QUERY_INDEX,        // 查詢大盤指數
    QUERY_HOT_STOCKS,   // 熱門股票/漲停板
    QUERY_HISTORY,      // 歷史K線數據
    QUERY_FINANCIALS,   // 財務數據/基本面
    QUERY_SECTOR,       // 板塊行情
    COMPARE_STOCKS,     // 對比分析
    TECHNICAL_ANALYSIS, // 技術分析
    UNKNOWN             // 不包含股票意圖
}

// 意圖解析結果
data class IntentResult(
    val intent: StockIntent,
    val stockCodes: List<String>,       // 提取出的股票代碼
    val stockNames: List<String>,       // 提取出的股票名稱
    val confidence: Float,              // 置信度 0.0~1.0
    val rawQuery: String,               // 原始用戶輸入
    val parsedParams: Map<String, Any>  // 額外參數（如時間範圍等）
)

// 意圖處理器接口
interface IntentHandler {
    // 檢查是否匹配此處理器
    fun match(input: String): Boolean
    // 解析意圖
    fun parse(input: String): IntentResult
    // 下一個處理器（Chain of Responsibility）
    var next: IntentHandler?
}

// 具體處理器鏈
class IntentProcessorChain {
    private val handlers = listOf(
        StockCodeHandler(),        // 純數字代碼 "600519"
        StockNameHandler(),        // 股票名稱 "茅台"
        HotStockHandler(),         // 熱門/漲停 "今天熱門股票"
        IndexHandler(),            // 指數 "上證指數"
        MultiStockHandler(),       // 多股票 "茅台和五糧液"
        AiIntentHandler(),         // 需要AI解析的意圖（最後一關）
    )
    
    fun process(input: String): IntentResult {
        for (handler in handlers) {
            if (handler.match(input)) {
                return handler.parse(input)
            }
        }
        return IntentResult(UNKNOWN, ...)
    }
}
```

class StockCodeHandler : IntentHandler {
    override fun match(input: String): Boolean {
        return STOCK_CODE_REGEX.containsMatchIn(input)
    }
    override fun parse(input: String): IntentResult {
        val codes = STOCK_CODE_REGEX.findAll(input)
            .map { it.value }
            .toList()

            stockCodes = codes,
            confidence = if (codes.size == 1) 0.95f else 0.85f
        )
    }
}

class StockNameHandler : IntentHandler {

    private val nameIndex = StockNameIndex() // 股票名稱索引

            intent = when {
                matched.any { it.isIndex } -> StockIntent.QUERY_INDEX
                else -> StockIntent.QUERY_PRICE
            },
            stockCodes = matched.map { it.code },
            stockNames = matched.map { it.name },
            confidence = 0.8f
        )
    }
}

### 3.4 數據倉儲（Repository）

```kotlin
class StockRepository(
    private val primarySource: StockDataSource,
    private val fallbackSources: List<StockDataSource>,
    private val cache: StockCache
) {
    fun getRealtime(codes: List<String>): Map<String, StockRealtime> {
        // 1. 先查緩存
        val cached = cache.get(codes)
        val uncached = codes - cached.keys
        
        if (uncached.isEmpty()) return cached
        
        // 2. 主數據源查詢
        try {
            val fresh = primarySource.fetchRealtime(uncached)
            cache.put(fresh)
            return cached + fresh
        } catch (e: Exception) {
            // 3. 主源失敗，降級到備用源
            for (fallback in fallbackSources) {
                try {
                    val fresh = fallback.fetchRealtime(uncached)
                    cache.put(fresh)
                    return cached + fresh
                } catch (_: Exception) { }
            }
            // 4. 全部失敗，返回緩存數據（即使過期）
            return cached
        }
    }
}
```

### 3.5 緩存層

```kotlin
class StockCache(private val ttlMs: Long = 3000) { // 默認3秒
    private val cache = LinkedHashMap<String, CacheEntry>(100)
    
    data class CacheEntry(
        val data: StockRealtime,
        val timestamp: Long
    )
    
    fun get(codes: List<String>): Map<String, StockRealtime> {
        val now = System.currentTimeMillis()
        return codes.mapNotNull { code ->
            val entry = cache[code]
            if (entry != null && (now - entry.timestamp) < ttlMs) {
                code to entry.data
            } else {
                null
            }
        }.toMap()
    }
    
    fun put(data: Map<String, StockRealtime>) {
        val now = System.currentTimeMillis()
        data.forEach { (code, realtime) ->
            cache[code] = CacheEntry(realtime, now)
        }
    }
}
```

### 3.6 服務層（核心門面）

```kotlin
class StockService(
    private val intentProcessor: IntentProcessorChain,
    private val repository: StockRepository,
    private val dataFormatter: StockDataFormatter,
    private val aiAnalyzer: AiStockAnalyzer?
) {
    /**
     * 處理用戶輸入，返回要注入到 prompt 的數據
     *
     * @param userMessage 用戶輸入
     * @return StockContext 包含意圖和格式化數據
     */
    fun processUserInput(userMessage: String): StockContext {
        // 1. 意圖識別
        val intent = intentProcessor.process(userMessage)
        
        // 2. 根據意圖獲取數據
        val data = when (intent.intent) {
            StockIntent.QUERY_PRICE -> {
                repository.getRealtime(intent.stockCodes)
            }
            StockIntent.QUERY_HOT_STOCKS -> {
                fetchHotStocks()
            }
            // ... 其他意圖
        }
        
        // 3. 格式化輸出
        val formattedData = dataFormatter.format(intent, data)
        
        return StockContext(
            intent = intent,
            hasStockData = data.isNotEmpty(),
            promptPrefix = formattedData
        )
    }
    
    // 註：複雜意圖由 AiStockAnalyzer 處理
    // 比如 "分析茅台走勢" → AI 決定調用歷史K線 → 分析

## 4. 完整數據流程

### 4.1 簡單查詢（無需 AI 解析）

    │
    ├─ StockNameHandler.match("茅台") → true

## 6. 格式化器設計

```kotlin
class StockDataFormatter {
    /**
     * 根據意圖和數據生成適合注入 prompt 的文本
     */
    fun format(intent: IntentResult, data: Any): String {
        return when (intent.intent) {
            StockIntent.QUERY_PRICE -> formatRealtime(intent, data as Map<String, StockRealtime>)
            StockIntent.QUERY_HOT_STOCKS -> formatHotStocks(data as List<StockRealtime>)
            StockIntent.QUERY_INDEX -> formatIndex(data as Map<String, StockRealtime>)
            StockIntent.COMPARE_STOCKS -> formatComparison(intent, data as Map<String, StockRealtime>)
            StockIntent.TECHNICAL_ANALYSIS -> formatTechnicalAnalysis(data as String)
            else -> ""
        }
    }
    
    private fun formatRealtime(intent: IntentResult, data: Map<String, StockRealtime>): String {
        val sb = StringBuilder()
        sb.appendLine("【實時行情數據】")
        sb.appendLine("（以下是用戶查詢的股票實時行情，請基於這些數據回答）")
        sb.appendLine()
        
        for ((code, stock) in data) {
            val arrow = if (stock.changeAmount > 0) "📈" else if (stock.changeAmount < 0) "📉" else "➡️"
            sb.appendLine("$arrow ${stock.name} ($code)")
            sb.appendLine("   當前價: ${stock.price} 元")
            sb.appendLine("   漲跌: ${if (stock.changeAmount > 0) "+" else ""}${stock.changeAmount} (${stock.changePercent}%)")
            sb.appendLine("   最高: ${stock.high}  最低: ${stock.low}")
            sb.appendLine("   成交量: ${stock.volume / 10000} 萬手")
            sb.appendLine("   昨收: ${stock.yestClose}  開盤: ${stock.open}")
            sb.appendLine("   更新時間: ${formatTimestamp(stock.timestamp)}")
            sb.appendLine()
        }
        
        return sb.toString()
    }
    
    private fun formatTimestamp(timestamp: Long): String {
        val sdf = java.text.SimpleDateFormat("HH:mm:ss", java.util.Locale.getDefault())
        return sdf.format(java.util.Date(timestamp))
    }
}

### 13.1 AKShare（⭐19k+）— 最成熟的 A 股數據庫

項目地址：https://github.com/akfamily/akshare

AKShare 是目前 GitHub 上最受歡迎的中國金融數據庫，其架構值得深度參考：

#### AKShare 的架構特點

```
akshare/
├── stock/
│   ├── cons.py                        # 所有 API URL 常量集中管理
│   ├── stock_zh_a_sina.py            # 新浪 A 股實時行情
│   ├── stock_zh_a_tx.py              # 騰訊 A 股實時行情  
│   ├── stock_zh_a_tick_tx.py         # 騰訊分筆數據
│   ├── stock_zh_a_special.py         # 特殊處理
│   ├── stock_hot_rank_em.py          # 東方財富熱門排行
│   ├── stock_hot_up_em.py            # 東方財富漲停板
│   ├── stock_intraday_em.py          # 東方財富分時圖
│   ├── stock_board_industry_em.py    # 東方財富行業板塊
│   ├── stock_board_concept_em.py     # 東方財富概念板塊

└── ...
```

**文件名命名規則**：`{資產類別}_{市場}_{數據源}.py`
- `stock_zh_a_sina` = 股票(A股) + 中國(A) + 新浪
- `stock_hot_rank_em` = 股票 + 熱門排行 + 東方財富
- `stock_board_industry_em` = 股票 + 板塊 + 行業 + 東方財富

#### 借鑒 AKShare 的關鍵設計

| AKShare 的做法 | 我們可以借鑒 |
|---------------|-------------|
| **`cons.py` 集中管理 URL** | 在 `data/sources/` 中每個 Source 自帶 URL，但 URL 常量統一放在 `StockApiUrls.kt` |
| **按數據源分文件** | 每個數據源獨立文件（`SinaStockSource.kt`, `TencentStockSource.kt`） |
| **按功能域分目錄** | `intent/`, `data/`, `formatter/`, `analysis/` 按職責分離 |
| **命名規則清晰** | 類名 + Source 後綴，一眼看出數據來源 |
| **異常靜默處理** | 數據源失敗時優雅降級，不影響主流程 |
| **支持批量查詢** | 新浪 API 原生支持多代碼逗號分隔 |

#### AKShare 沒有解決的問題（我們需要自己做的）

- **沒有意圖識別層** — AKShare 是查詢庫，用戶需要自己決定調哪個函數
- **沒有 AI 集成** — AKShare 不涉及 LLM 調用
- **沒有緩存層** — 每次都是直接調 API

### 13.6 關鍵設計決策（參考開源項目後）

1. **主數據源：新浪財經**（參考 AKShare 的 `stock_zh_a_sina`）
   - 免費、無需 token、查詢穩定
   - 支持批量查詢（逗號分隔）

2. **備用數據源：騰訊財經 + 東方財富**（參考 AKShare 的 `stock_zh_a_tx` + `stock_hot_rank_em`）
   - PHP 格式（騰訊），JSON 格式（東方財富）
   - 東方財富特別適合熱門排行、漲停板等數據

3. **文件名命名規則**（參考 AKShare）
   - `{角色}{數據源}Source.kt` — e.g., `SinaStockSource`
   - `{意圖}Handler.kt` — e.g., `HotStockHandler`

4. **錯誤處理策略**（參考 Tushare）
   - 統一返回 `StockResult<T>` sealed class
   - 主源失敗 → 自動降級到備源 → 全部失敗返回緩存

5. **集中式 URL 管理**（參考 AKShare 的 `cons.py`）
   - 新增 `StockDataUrls.kt` 文件，集中存放所有 API 地址常量

## 14. 設計模式總結

| 模式 | 使用場景 | 好處 |
|------|----------|------|
| **Chain of Responsibility** | IntentProcessorChain | 每個 Handler 只處理自己擅長的意圖，可插拔 |
| **Repository** | StockRepository | 統一數據訪問接口，隔離數據源變化 |
| **Strategy** | StockDataSource | 可隨時切換/添加數據源 |
| **Facade** | StockService | 簡化 ChatActivity 的調用 |
| **Null Object** | NotApplicable | 避免 null check |
| **Decorator** | Cache 包裹 Repository | 透明添加緩存能力 |
